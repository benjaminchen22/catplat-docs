catplat.database.ase module

This module consists of classes involved with the retrieval of information from the Database. Database contains many DatabaseEntry. Each DatabaseEntry has it’s own set of DatabaseAttributes QuerySet (main interface for users) -> one or multiple Query based on specified -> retrieve from database from att

update_from_relaxed_folder(unrelaxed_id, folder='.', overwrite=False, suppress_errors=False)[source]

Extracts the information from a specified relaxed calculation folder, and updates the relaxed entry(ies) if one or more already exist. If not, creates a new relaxed entry. Assumes the unrelaxed entry with the specified unrelaxed_id corresponding to the calculation folder already exists in the db, and inherits the attributes from there. Returns a list of tuples where each tuple summarizes the results pertaining to each relaxed entries found. Each tuple contains a short status message, the calculation path, the unrelaxed ID, and the relaxed ID. The possible status messages are: ‘stored’, ‘updated’, ‘overwritten’, ‘skipped’, ‘multiple_dir’, ‘no_unrelaxed’, ‘unconverged’, ‘atoms_unmatched’. The expected return format is: [tuple_1, tuple_2, ….], where tuple_n: (msg: str, path: str, unrelaxed_id: int, relaxed_id: int|None)

class catplat.database.ase.QuerySet(bulk=None, slab=None, adsorbate=None, other=None)[source]

Bases: object

Main class that interacts and stores user specifications.

The QuerySet contains 4 main SpecificationsSet classes, namely BulkSpecificationsSet, SlabSpecificationsSet, AdsorbateSpecificationsSet and OtherSpecificationsSet. The attributes of the SpecificationsSet classes are iterables such as lists/tuples. These 4 SpecificationsSet classes stores all the user input attributes to run the catplat workflow.

bulk

Class containing the bulk attributes provided by the user. More details in catplat.specifications.bulk.py

Type:

BulkSpecificationsSet

slab

Class containing the slab attributes provided by the user. More details in catplat.specifications.slab.py

Type:

SlabSpecificationsSet

adsorbate

Class containing the adsorbate attributes provided by the user. More details in catplat.specifications.adsorbate.py

Type:

AdsorbateSpecificationsSet

others

Class containing the other attributes provided by the user. More details in catplat.specifications.other.py

Type:

OtherSpecificationsSet

Examples

Attributes of QuerySet can be set after instantiation: i.e., qs = QuerySet()

qs.bulk.chemsys = [‘Cu’]

or through a dictionary and factory classmethod: i.e., bulk_attributes = {‘chemsys’: [Cu]}

qs = QuerySet.from_attr(bulk=bulk_attributes)

catplat.database.mysql module

catplat.database.mysql.create_select_statement(self, keys, cmps, sort=None, order=None, sort_table=None, what='systems.*')[source]

Monkeypatch for SQLite3Database.create_select_statement.

The select statement here is much faster as it recursively narrows down on the search space, whereas the original select statement searches the database n times if there are n comparisons. A speedup of more than 100x can seen for databases with MySQL backend. Not tested if .db files (sqlite3 backend) have any speedup.

sort,`sort_table`, and order arguments are not supported yet.

Update Mar 16 2023: seems to be slow for large databases
