test_adsorbates_not_recalculated_if_binding_site_changes(tmpdir, fake_vasp_jobrunner, bond_length_buffer, expected_result)[source]

Test on 4-fold hollow site of 211 surface as it has a non-normal adsorption vector that will change the x-y adsorption coordinates if the bond length is changed.

If using original buffer of 0, then we expect catplat to recognize that the new adsorbate generated during recalculation are the same as the old one, and only one result is produced.

If using slightly different buffer of 2, then the new adsorption coords are slightly different from the old one (smaller than the specified tolerance), so it will NOT trigger a recalculation and only one result is produced.

If using large new buffer of 10, then the new adsorption coords are very much different from the old one (greater than the specified tolerance), so it will trigger a recalculation and more results are produced.
